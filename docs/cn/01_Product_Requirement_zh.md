# 1. 引言

## 1.1 背景
一级方程式（F1）车迷会从各类渠道获取信息，包括国际汽联（FIA）官方文件、F1 官方网站、车队公告、技术分析、计时服务以及赛车媒体。

尽管大语言模型能够回答 F1 相关的通用问题，但无法可靠地提供以下内容：
- 实时新闻资讯
- 官方赛事信息
- 准确的规则解读
- 可溯源的 FIA 文档引用

PitWall 智能助手通过将大语言模型与外部工具、检索增强生成（RAG）相结合，解决上述问题。

系统并非仅依靠模型知识生成答案，而是先调取权威信息，再进行推理作答。

---

## 1.2 产品定位
PitWall 智能助手是一款专为一级方程式打造的生产级 AI 助手。

它融合了以下能力：
- 自然语言交互
- 智能工具编排
- 实时信息检索
- FIA 规则检索
- 上下文感知推理

产品同时面向普通车迷与技术向用户。

---

## 1.3 产品目标
项目的核心目标包括：
- 提供可靠的一级方程式信息
- 减少错误信息与模型幻觉
- 降低 FIA 规则的查阅门槛
- 简化赛事信息的获取路径
- 展示生产级 AI 工程实践能力

# 4. 功能范围
PitWall 智能助手 1.0 版本包含五大核心能力领域：
- 新闻检索
- 赛事信息
- 规则问答
- 策略分析
- 对话交互

各项能力的详细说明如下。

---

# 5. 功能需求

## FR-001
### 对话交互
**优先级**
P0

**描述**
系统需支持自然语言对话能力。

**验收标准**
- 支持多轮对话
- 具备上下文感知能力
- 支持流式响应输出
- 支持 Markdown 格式渲染

---

## FR-002
### 新闻检索
**优先级**
P0

**描述**
系统需从可信外部源获取最新的一级方程式新闻。

**输入**
自然语言提问

**示例**
“今天发生了什么？”

**输出内容**
- 新闻摘要
- 信息来源
- 发布时间
- 关联车队/车手

**验收标准**
- 10 秒内生成响应
- 支持多条新闻内容汇总
- 信息均来自外部权威提供商

---

## FR-003
### 赛事信息
**优先级**
P0

**描述**
系统需调取官方赛事信息。

**支持的数据类型**
- 全年赛历
- 单站比赛周末日程
- 排位赛成绩
- 冲刺赛成绩
- 正赛成绩排名
- 车手积分榜
- 车队积分榜

**示例**
“下一站大奖赛什么时候开始？”

**验收标准**
信息必须与最新的官方公开数据保持一致。

---

## FR-004
### FIA 规则问答
**优先级**
P0

**描述**
系统需通过检索增强生成（RAG）技术，解答 F1 赛事运动规则与技术规则相关问题。

系统不会仅依赖语言模型作答，而是先检索相关规则章节，再生成答案。

**示例问题**
- 什么是封闭检车区（Parc Fermé）？
- 什么是维修站不安全放行？
- 赛车底板的最小厚度要求是多少？
- 车队什么时候可以更换动力单元？

**输入**
自然语言提问

**输出内容**
响应需包含：
- 问题答案
- 支撑规则原文
- 规则条款编号
- 文档标题
- 置信度（内部使用）

**验收标准**
- 可检索到相关的文档片段
- 检索到的上下文会融入最终回答
- 通过基于事实的生成，最大限度减少幻觉内容

---

## FR-005
### 策略分析
**优先级**
P1

**描述**
系统需结合检索到的赛事信息与大语言模型推理能力，提供赛事策略分析。

该功能侧重解释已有决策，而非预测结果。

**示例问题**
- 迈凯伦为什么提前进站？
- 法拉利为什么丢失了场上位置？
- 这次安全车出动对谁更有利？

**输出内容**
系统利用已有的赛事信息与规则知识，解释相关战术决策。

**验收标准**
- 分析内容逻辑清晰、结构完整
- 推理过程引用已有的赛事上下文
- 系统明确区分事实信息与分析内容

---

## FR-006
### 对话记忆
**优先级**
P1

**描述**
系统需在单次用户会话中维护对话上下文。

用户提出追问时，无需重复此前的问题背景。

**示例**
用户：
> 维斯塔潘为什么被处罚？

追问：
> 这个判罚和之前的比赛标准一致吗？

系统需能识别追问指向的是此前讨论的赛事事件。

**验收标准**
- 对话上下文完整保留
- 追问内容可被正确理解与响应
- 不同用户的会话状态相互隔离

---

## FR-007
### 引用来源支持
**优先级**
P1

**描述**
当答案基于检索到的文档生成时，响应需提供对应的支撑参考文献。

可引用的要素包括：
- 文档标题
- 条款编号
- 章节标题
- 页码

**验收标准**
- 有可用来源时必须展示引用信息
- 引用内容与检索到的文档一一对应
- 引用格式保持统一规范

---

## FR-008
### 错误处理
**优先级**
P1

**描述**
系统需优雅处理检索、工具执行、语言模型调用过程中的各类故障。

**可能的故障场景**
- 外部 API 不可用
- 向量数据库超时
- 文档缺失
- 大语言模型请求失败

**验收标准**
- 用户会收到清晰易懂的错误提示
- 内部错误会被完整记录日志
- 应用整体保持可用，不会因局部故障崩溃

---

# 8. 系统约束
初始版本存在以下有意设定的限制。

## 数据范围
仅支持一级方程式（F1）赛事。
F2、F3、FE、世界耐力锦标赛（WEC）、世界拉力锦标赛（WRC）等其他 FIA 旗下赛事，均不在 1.0 版本范围内。

---

## 语言支持
1.0 版本仅提供英文回答。
多语言支持将在未来版本中考虑。

---

## 知识来源范围
基于规则的回答仅依赖已录入知识库的文档。
未被索引的文档无法被引用。

---

## 外部依赖
系统依赖多项外部服务。
例如：
- 大语言模型服务商
- 新闻提供商
- 赛事信息 API
- 向量数据库

上述服务的临时故障可能影响部分功能。

---

## 实时性说明
赛事信息与新闻的准确性取决于外部数据提供商的更新频率。
系统不保证秒级实时同步。

---

# 9. 假设条件
开发过程基于以下假设：
- 用户具备网络连接条件
- 外部 API 可正常运行
- FIA 文档会定期更新
- 嵌入模型与已索引的数据保持兼容
- 所支持的大语言模型具备工具调用能力

若任一假设发生变化，可能需要对实现方案进行对应调整。

# 12. 风险
项目规划阶段已识别以下风险。

---

## R-001
### 外部 API 可用性
部分功能依赖第三方服务。

**潜在影响**
- 新闻内容缺失
- 赛事数据缺失
- 部分功能不可用

**应对措施**
- 超时处理机制
- 重试机制
- 功能优雅降级

---

## R-002
### 大语言模型幻觉
语言模型可能生成错误信息。

**应对措施**
- 检索增强生成（RAG）
- 基于工具的推理模式
- 引用来源支持
- 基于事实的提示词设计

# 15. 结语
PitWall 智能助手致力于打造一款准确、可解释、面向生产环境的一级方程式 AI 助手。

产品将对话式 AI 与智能工具编排、检索增强生成相结合，输出基于权威来源、值得信赖的答案。

1.0 版本重点搭建扎实的工程基础，同时交付以下实用能力：
- F1 新闻检索
- 赛事信息查询
- FIA 规则问答
- 策略分析
- 多轮对话交互

本文档定义的需求，将作为系统架构设计、开发实现、测试验证与后续迭代的基准依据。
